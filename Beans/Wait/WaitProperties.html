Component Wait
Wait
Component Level: High
Category: SW-User Components
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • SDK - Link to the SDK
  • Manual Clock Values - In case the bean cannot get the clock/speed settings from the CPU bean, you may specify here manual values.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • CPU_BUS_CLK_HZ - Bus Clock in Hz
    • CPU_BUS_CLK_HZ_HIGH - Bus Clock in Hz
    • CPU_BUS_CLK_HZ_LOW - Bus Clock in Hz
    • CPU_BUS_CLK_HZ_SLOW - Bus Clock in Hz
  • Delay100usFunction - Function which does a 100us delay. Typically provided by the CPU bean (you need to enable it).
  • RTOS - If using an RTOS, then non blocking functions can be used. Otherwise they are routed to blocking methods.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • RTOS - RTOS to be used
  • Watchdog - If during wait a watchdog shall be kicked
    The following items are available only if the group is enabled (the value is "Enabled"):
    • non-LDD - Used for non-LDD (Logical Device Driver) components, e.g. for S08 or ColdFire.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Watchdog - If a watchdog shall be kicked during waiting
    • LDD - Enabled for architectures with LDD (Logical Device Drivers), e.g. Kinetis.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Watchdog - Interface to Watchdog driver
