Bean Shell
Command Line Shell
Component Level: High
Category: Software-User Components
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Echo - If enabled, the shell will echo characters read on the terminal. The shell offers the echo command to turn echo on and off.
  • Prompt - Prompt string to be used. Make sure the string is in double quotes. You can use normal C escape characters like \r or
    as well.
  • Project Name - Here you can specify a project name which is shown if you use the help command.
  • Silent Mode Prefix - If commands start with this prefix character, the command will be executed and no prompt will be written. To disable silent support (for better code efficiency), use an empty char.
  • Blocking Send - If enabled, then sending using SendChar() to the communication channel will block until success.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Wait - Interface to waiting routine
    • Timeout (ms) - Timeout used to wait. Use zero for blocking send.
    • Wait Time (ms) - Waiting time in milli seconds if sending was blocked (output buffer full). Use 0 for polling mode.
    • RTOS Wait - If set to yes, then it uses the RTOS waiting/delay instead of busy waiting if RTOS is present.
  • Status Colon Pos - Position of the ':' for the status commands. Used for SendStatusString().
  • Help Semicolon Pos - Position of the ';' in the help commands. Used for SendHelpString().
  • Multi Command - Enables execution of multiple commands in a single command, separated with a separation char.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Length - Length of possible sub command string. Used as buffer on the stack, so should not be choosen too large
    • Separator - Multiple commands can be separated using this separation character
  • Utility - Interface to utility
  • Default Serial - If enabled, the Shell implements a default internal serial interface. Otherwise the SetStdIO() function can be used to set one.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Console Interface - Interface to low level asynchroserial component
  • Semaphore - If enabled and an RTOS is present, a semaphore is used to grant mutual access to the console/serial/SCI hardware. Useful if multiple tasks needs to write to the shell console. Make sure you call the Init() routine of the componenent first in order to create the semaphore.
  • Critical Section - Interface to critical section handling
  • History - If enabled, the component will store up to four entered commands. Recall them by enter a non-alphanumeric key.
    The following items are available only if the group is enabled (the value is "yes"):
    • Number of History Items - Defines the number of history items.
    • Stored Characters - Stored caracters for each history line.
    • Char for Next - Character key for previous item in history. Use \t for TAB and \e for SHIFT+TAB
    • Char for Previous - Character key for previous item in history. Use \t for TAB and \e for SHIFT+TAB
  • Kinetis SDK - Interface to Kinetis SDK to ensure additional symbols are defined in case Kinetis SDK is used.
