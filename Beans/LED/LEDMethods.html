Bean LED
Driver for a LED
Component Level: Low
Category: CPU External Devices-Display
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • On - This turns the LED on.

      ANSIC prototype: void On(void)

  • Off - This turns the LED off.

      ANSIC prototype: void Off(void)

  • Neg - This negates/toggles the LED

      ANSIC prototype: void Neg(void)

  • Get - This returns logical 1 in case the LED is on, 0 otherwise.

      ANSIC prototype: uint8_t Get(void)
    • Return value:uint8_t - Status of the LED (on or off)

  • Put - Turns the LED on or off.

      ANSIC prototype: void Put(uint8_t val)
    • val:uint8_t - value to define if the LED has to be on or off.

  • SetRatio16 - Method to specify the duty cycle. If using a PWM pin, this means the duty cycle is set. For On/off pins, values smaller 0x7FFF means off, while values greater means on.

      ANSIC prototype: void SetRatio16(uint16_t ratio)
    • ratio:uint16_t - Ratio value, where 0 means 'off' and 0xffff means 'on'

  • ParseCommand - Shell Command Line parser. This method is enabled/disabled depending on if you have the Shell enabled/disabled in the properties.

      ANSIC prototype: uint8_t ParseCommand(const unsigned char *cmd, bool *handled, const %@Shell@'ModuleName'%.StdIOType *io)
    • cmd: Pointer to uint8_t - Pointer to command string
    • handled: Pointer to bool - Pointer to variable which tells if the command has been handled or not
    • io: Pointer to uint8_t - Pointer to I/O structure
    • Return value:uint8_t - Error code

  • Deinit - Deinitializes the driver

      ANSIC prototype: void Deinit(void)

  • Init - Performs the LED driver initialization.

      ANSIC prototype: void Init(void)


Note: Some methods can be implemented as macros.
