Component SimpleEvents
Simple Event Handling
Component Level: High
Category: SW-User Components
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • SetEvent - Sets an event (number) be processed by the next HandleEvent() call

      ANSIC prototype: void SetEvent(uint8_t event)
    • event:uint8_t - The event (number) to be set so it will be called by the HandleEvent() routine later on. Note that there cannot be multiple instances of an event, and the lower the event number, the higher the priority.

  • ClearEvent - Clears one event (number). If the event is not set, then the function has no effect.

      ANSIC prototype: void ClearEvent(uint8_t event)
    • event:uint8_t - The event number to be cleared.

  • EventsPending - Returns true if any events are pending, false otherwise

      ANSIC prototype: bool EventsPending(void)
    • Return value:bool - True in case any events are pending, false otherwise.

  • GetEvent - Allows to check if an event is set or not.

      ANSIC prototype: bool GetEvent(uint8_t event)
    • event:uint8_t - The event number to check. If the event is set, the function returns TRUE.
    • Return value:bool - none

  • GetClearEvent - Allows to check if an event is set or not. If set, it will be automatically cleared.

      ANSIC prototype: bool GetClearEvent(uint8_t event)
    • event:uint8_t - The event number to check. If the event is set, the function returns TRUE.
    • Return value:bool - none

  • HandleEvent - Event handler, to be called periodically.

      ANSIC prototype: void HandleEvent(void)


Note: Some methods can be implemented as macros.
