Component GenericSWSPI
Software synchronous serial communication
Component Level: High
Category: CPU External Devices-Communication
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • RecvChar - If any data received, this method returns one character, otherwise it returns an error code (it does not wait for data).

      ANSIC prototype: uint8_t RecvChar(uint8_t *Chr)
    • Chr: Pointer to uint8_t - A pointer to the received character.
    • Return value:uint8_t - Error code, possible codes:
      ERR_OK - OK
      ERR_RXEMPTY - No data in receiver
      ERR_OVERRUN - Overrun error is detected

  • SendChar - Sends one character to the channel.

      ANSIC prototype: uint8_t SendChar(uint8_t val)
    • val:uint8_t - Character to send.
    • Return value:uint8_t - Error code, possible codes:
      ERR_OK - OK

  • CharsInRxBuf - Returns number of characters in the input buffer.

      ANSIC prototype: uint8_t CharsInRxBuf(void)
    • Return value:uint8_t - Returns number of characters in the input buffer.

  • CharsInTxBuf - Returns number of characters in the output buffer.

      ANSIC prototype: uint8_t CharsInTxBuf(void)
    • Return value:uint8_t - Returns number of characters in the output buffer.

  • SetShiftClockPolarity - Sets the shift clock polarity at runtime. Output data will be shifted on the selected edge polarity.

      ANSIC prototype: uint8_t SetShiftClockPolarity(uint8_t Edge)
    • Edge:uint8_t - Edge polarity, possible values:
      0-falling edge,
      1-rising edge.
    • Return value:uint8_t - Error code, possible codes:
      ERR_OK - OK

  • SetIdleClockPolarity - Set the idle clock polarity at runtime. If communication does not run, the clock signal will be set to the required level.

      ANSIC prototype: uint8_t SetIdleClockPolarity(uint8_t Level)
    • Level:uint8_t - Idle clock polarity:
      0-low
      1-high
    • Return value:uint8_t - Error code, possible codes:
      ERR_OK - OK

  • Write_ReadDummy - Writes to the bus, but does not read.

      ANSIC prototype: void Write_ReadDummy(uint8_t val)
    • val:uint8_t - value to put on the bus

  • SetSlowMode - Changes to slow mode

      ANSIC prototype: void SetSlowMode(void)

  • SetFastMode - Sets the fast mode

      ANSIC prototype: void SetFastMode(void)


Note: Some methods can be implemented as macros.
